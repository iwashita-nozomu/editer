# 専用コードエディター要件（v0.1 最小核）

あなたの最新意図を反映し、**まずは核機能だけ**に絞った仕様。

## 0. 今回の結論（スコープを絞る）
- v0.1は **ウィンドウ管理 + ディレクトリ管理画面 + 編集 + シェル** に集中
- GUIは「最低限動く核」優先（見た目は後で拡張）
- 起動はCLI（`myeditor <path>`）
- 対応OS: Windows / macOS / Linux
- 実装言語: C++23（最低 C++20）

---

## 1. v0.1の中核機能

### 1.1 ウィンドウ管理
- 分割（上下/左右）
- フローティング
- マルチモニター配置
- ウィンドウ表示名（ユーザー命名）
- レイアウト保存/復元

### 1.2 ディレクトリ管理画面
- マルチルート（複数ディレクトリを1ワークスペース）
- ツリー表示、ルートごとの絞り込み検索
- ルート別にタスク/除外設定を保持

### 1.3 重複オープン通知（新規要件）
- 同じファイルを既に開いている場合:
  - 「既存タブへ移動 / 新規ビューで開く」を通知ダイアログで選択
- 同じディレクトリを既にワークスペース登録済みの場合:
  - 「既存ルートへフォーカス / 別名で登録（非推奨）」を通知
- 通知には「次回から同じ動作を適用」チェックを用意

### 1.4 編集・スクロール操作
- シンタックスハイライト付き編集
- 高速スクロール対応:
  - マウスホイール通常スクロール最適化
  - **ホイール押し込み（中ボタン）中の高速パン/オートスクロール**対応
- 大規模ファイルでもスクロール追従性を維持

### 1.5 シェル統合
- 1ペインの統合ターミナル（分割は将来拡張）
- Windows: ConPTY / Linux・macOS: PTY

---

## 2. GUI方針（最小で始める）
- 方針: **核機能を成立させる最低限GUIを先に作る**
- 自作GUIは可（楽しさ重視）だが、初期は以下を優先
  1. 入力安定性（IME/日本語）
  2. スクロール体感
  3. ウィンドウ/ペイン操作の一貫性
- デザイン強化（テーマ、装飾、アニメーション）はv0.2以降

---

## 3. 「何を開くか」の整理（現時点）
現時点で優先して開く対象:
1. ファイル
2. ディレクトリ（ワークスペース）
3. シェル

追加候補（今は必須にしない）:
- 検索結果ビュー
- 差分ビュー（git diff）
- 実行ログ/タスク出力

---

## 4. 設定で変更可能にする項目
- ウィンドウ色/パネル色
- ウィンドウ表示名
- 重複オープン時の既定動作
- コマンド実行結果の表示先（同一パネル/別ウィンドウ）
- キーバインド

設定ファイルは `settings.toml` を使用。

---

## 5. ビルド要件
- 初回コンパイルは**1コマンド必須**
- 例: `cmake -S . -B build && cmake --build build -j`
- もしくは `make build`

---

## 6. 永続化/復旧
- 一次情報はローカル専用ディレクトリへ保存
- 保存対象: レイアウト、ワークスペース、履歴、セッション
- 異常終了後にセッション復元可能

---

## 7. 非機能目標
- 起動: 500ms以内（初回除く）
- 10万行級でも実用スクロール
- 中ボタン高速パン時の入力遅延を最小化
